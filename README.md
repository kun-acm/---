# 基础医学概论刷题

一个纯前端、零依赖的基础医学刷题 + 知识点学习工具，涵盖**解剖学、生理学、组织学**三门学科。工程化多文件结构，可直接部署到 GitHub Pages。

---

## 📦 项目结构

```
基础医学概论刷题/
├── index.html              # 页面骨架（引用外置 CSS/JS）
├── css/
│   └── style.css           # Claude 风格样式
├── js/
│   ├── data-anatomy.js     # 解剖学题库（5 个题库，155 题）
│   ├── data-physiology.js  # 生理学题库（9 章，375 题）
│   ├── data-histology.js   # 组织学题库（4 组，113 题）
│   ├── data-knowledge.js   # 三门学科知识点
│   └── app.js              # 主逻辑（渲染/判分/状态/导航/知识点页）
└── README.md
```

> ⚠️ 数据文件用 `<script src>` 加载（非 fetch），因此**双击 index.html 本地打开也能正常使用**，无需启动服务器。

## 🎯 功能

### 刷题（三门学科共 643 题）

| 功能 | 说明 |
|------|------|
| **学科筛选** | 单一题库模式下切换解剖学/生理学/组织学 |
| **单一题库模式** | 按原顺序逐题刷，下拉切换题库（18 个题库） |
| **全题库乱序模式** | 643 题全部打乱，每次"重新开始"重新洗牌 |
| **错题练习模式** | 只练未掌握的错题，做对自动标记"已掌握" |
| **即时判分** | 点击选项立刻显示对错 + 详细解析 |
| **整组判分** | 全部做完统一提交，跳过题计"未答"不计分 |
| **断点续做** | 每模式独立记录进度，关闭浏览器再打开接着做 |
| **正确率统计** | 总体正确率 + 各题库正确率 + 完成度进度条 |
| **错题本** | 去重、累计错误次数、做对标记已掌握、点击跳转 |
| **键盘操作** | `←` `→` 切换题目，数字键 `1`-`5` 选选项（支持 5 选项题） |

### 知识点学习（新增）

| 功能 | 说明 |
|------|------|
| **类目录式浏览** | 左侧学科 tab + 章节/系统列表，右侧内容面板 |
| **三门学科分开** | 解剖学 / 生理学 / 组织学 独立 tab |
| **要点高亮** | 必考点（琥珀色）、易错点（红色）特殊标记 |
| **响应式** | 移动端左栏自动折叠 |

## 🚀 使用方法

### 本地使用

**双击** `index.html`，用浏览器打开即可。推荐 **Chrome** 或 **Edge**。

> 如遇本地打开进度无法保存（部分浏览器对 `file://` 的 localStorage 限制），可启动本地服务器：
> ```bash
> python -m http.server 8080
> # 浏览器访问 http://localhost:8080/
> ```

### 部署到 GitHub Pages

1. 在 GitHub 新建仓库（Public 或 Private 均可）
2. 把整个 `基础医学概论刷题` 文件夹内容上传到仓库根目录（`index.html` 必须在根目录）
3. 仓库 **Settings → Pages**
4. **Source** 选 `Deploy from a branch`，Branch 选 `main` / `(root)`，点 **Save**
5. 等待 1-2 分钟，访问页面给出的 `https://<用户名>.github.io/<仓库名>/`

> ⚠️ 仓库根目录必须有 `index.html`，GitHub Pages 默认服务该文件。若打开网址显示 README，说明 `index.html` 不在根目录或文件名不对。

## ⚠️ 重要提示

### 答案准确性

所有答案和解析均含 AI 生成成分，**不能保证 100% 正确**：
- 解剖学：AI 生成答案与解析
- 生理学：文档答案 + AI 审核更正 + AI 解析（文档与 AI 不符时以 AI 为准）
- 组织学：完全 AI 生成

建议：将答案视为"参考答案"，如有疑问请以教材为准。

### 数据存储

- 答题记录、进度、错题本均存储在浏览器 **localStorage**
- 清除浏览器缓存 / 无痕模式 / 换浏览器 / 换电脑 会导致数据丢失
- 数据存储在本地，不上传任何服务器

### 题库数据结构

每个数据文件导出一个全局数组（`BANKS_ANATOMY` / `BANKS_PHYSIOLOGY` / `BANKS_HISTOLOGY`），由 `app.js` 合并为 `QUIZ_BANKS`：

```javascript
const BANKS_PHYSIOLOGY = [
  {
    id: "ch1",              // 题库唯一 ID
    subject: "生理学",       // 学科（用于筛选）
    name: "第一章 绪论",     // 显示名称
    questions: [
      {
        stem: "神经调节的基本方式是：",   // 题干
        options: ["反射","反应","适应","正反馈调节","负反馈调节"],  // 选项数组
        answer: 0,          // 正确答案索引（0=A, 1=B, ... 4=E）
        exp: "反射是神经调节的基本方式……"  // 解析
      }
    ]
  }
];
```
